name: Frontend Components
description: Design and build reusable, composable UI components following best practices for single responsibility, encapsulation, props design, and state management. Use this skill when creating or modifying front-end components in any framework. Use when working on files like *.jsx, *.tsx, *.vue, *.svelte, components/*.py, or any file that defines a UI component. Use when designing component APIs and props, splitting large components, lifting state, composing components together, or establishing naming conventions for components.
---

## When to use this skill:

- When creating new UI components
- When refactoring or splitting existing components
- When working on files such as *.jsx, *.tsx, *.vue, *.svelte, or component definition files
- When designing component props and default values
- When deciding where state should live (local vs. lifted)
- When composing smaller components into larger UI structures
- When naming components and establishing component conventions
- When reducing the number of props on an overly complex component
- When documenting component usage and examples
- When encapsulating internal implementation details within a component
