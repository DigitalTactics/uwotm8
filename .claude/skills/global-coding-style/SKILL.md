## When to use this skill:

- When writing or editing any source code file in the project
- When naming variables, functions, classes, modules, or files
- When refactoring code to eliminate duplication (DRY principle)
- When splitting large functions into smaller, focused ones
- When removing dead code, unused imports, or commented-out blocks
- When formatting code for consistency (indentation, line breaks, spacing)
- When reviewing whether backward compatibility code is actually needed
- When ensuring descriptive, intent-revealing names are used throughout
