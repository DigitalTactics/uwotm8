## When to use this skill:

- When adding validation logic to form inputs or API endpoints
- When working on files such as validators.py, forms.py, schemas.py, serializers.py, or form components
- When implementing server-side validation for data integrity and security
- When adding client-side validation for user experience feedback
- When sanitizing user input to prevent SQL injection, XSS, or command injection
- When writing field-specific error messages for invalid input
- When implementing allowlist-based validation rules
- When validating business rules (date ranges, balance checks, permissions)
- When ensuring consistent validation across web forms, API endpoints, and background jobs
