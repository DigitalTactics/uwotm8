name: Global Commenting
description: Write minimal, helpful code comments that explain why, not what, while keeping code self-documenting through clear naming and structure. Use this skill when adding, editing, or reviewing comments in any source code file. Use when working on files like *.py, *.js, *.ts, *.jsx, *.tsx, *.rb, *.go, *.java, or any programming language file. Use when deciding whether a comment is necessary, writing docstrings, explaining complex logic sections, or cleaning up outdated or change-tracking comments.
---

## When to use this skill:

- When adding comments to any source code file
- When editing or reviewing existing code comments
- When deciding whether code needs a comment or should be made self-documenting instead
- When writing docstrings or function-level documentation
- When explaining large or complex sections of code logic
- When encountering comments that describe recent changes or fixes (these should be removed)
- When ensuring comments are evergreen and will remain relevant over time
