## When to use this skill:

- When creating new database migration files
- When modifying existing migrations or migration scripts
- When working on files in migrations/ directories, alembic/ directories, or similar migration folders
- When adding, removing, or altering database tables, columns, or indexes
- When writing rollback/down methods for migrations
- When planning zero-downtime deployment strategies for schema changes
- When separating schema migrations from data migrations
- When creating indexes on large tables
- When renaming or restructuring database objects
