name: Testing Test Writing
description: Write focused, behavior-driven tests that cover core user flows without over-testing edge cases or implementation details. Use this skill when creating or modifying test files, test functions, or test fixtures. Use when working on files like test_*.py, *_test.py, *.test.js, *.test.ts, *.spec.js, *.spec.ts, *_spec.rb, or any file in a tests/ or __tests__/ directory. Use when deciding what to test, writing test names, mocking external dependencies, structuring test suites, or determining the right level of test coverage for a feature.
---

## When to use this skill:

- When creating new test files or adding test functions
- When working on files such as test_*.py, *_test.py, *.test.js, *.test.ts, *.spec.js, *.spec.ts, or *_spec.rb
- When working in tests/, __tests__/, or spec/ directories
- When deciding which behaviors and user flows to test
- When writing descriptive test names that explain the scenario and expected outcome
- When mocking external dependencies (databases, APIs, file systems)
- When determining whether to test an edge case or defer it
- When structuring test suites for a new feature
- When keeping tests fast and focused on behavior rather than implementation
